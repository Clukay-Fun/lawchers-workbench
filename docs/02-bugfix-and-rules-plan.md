## 任务 A · 工作台复核流修复（lawchers-workbench）

### A1. 删除材料后"复活" / 显示"尚未预处理"
- 根因：删除后仍用旧 `currentCase` 调 `onSelect`，把已删材料写回前端状态；ReviewPanel 未清空旧状态，把失败的复核请求渲染成"预处理入口"。
- 修：
  - 删除后基于**最新**材料列表修正选中索引，不用旧 `currentCase` 回写已删材料。
  - `ReviewPanel` 在 `materialId` 变化时**彻底重置全部本地状态**（previewMd/manifest/decisions/revealed/selection/error 等）。
- 验证：删除当前选中材料 → 列表立即移除、不复活；中间区切到有效材料或空态，不再误显"开始预处理"。贴后端 `DELETE /api/materials/:id` 200。

### A2. `refreshReview` 不再吞错
- 现状吞掉异常，导致失败被掩盖成空/预处理页。
- 修：错误要 surface（toast/错误态），不静默 `console.warn` 了事。

### A3. `deleteDecision` 参数错误（手工高亮不能右键取消）
- 根因：后端 `deleteDecision(matId, d.id)` 仓储函数只接收一个 `id`，实际按材料 id 删，删错对象。
- 修：仓储函数按**决策 id**删除（修正参数与返回值）；手工新增后，前端直接用**后端返回的新决策对象**（含真实 id/origin），保证右键能精准定位删除。
- 验证：划词新增一处高亮 → 右键取消 → 该处且仅该处还原为普通文字；贴前后 `decisions`。

### A4. `exported` 不再禁止编辑
- 根因：导出后状态变 `exported`，但更新决策只允许 `reviewing/ready` → 导出后再编辑必 409。
- 修：导出**不应**把材料锁死。允许在 `exported` 下更新决策，且决策变更后回到 `reviewing`（导出结果信息改记在 `verification_status / redacted_path / audit`，不靠 `processing_status` 表达"导出过"）。
- 验证：导出后再右键某处 → 不报 409，状态回到 `reviewing`。

### A5. PDF 导出禁用，不给必失败按钮
- 根因：PDF 按决策导出后端写死 501（未实现），前端仍显示"导出脱敏副本"。
- 修：PDF 文档的导出入口显示「导出尚未支持」并**禁用**；仅 DOCX（及其它已支持格式）可导出。

### A6. DOCX 导出：最终确认弹窗 + 人工确认审计
- 去掉旧"完成复核"独立步骤；改为**每次导出弹确认框**，文案：
  > 本次将导出：N 处脱敏、M 处保留、K 处手工补标。还有 X 处自动标注未逐项确认，是否按当前状态统一确认并导出？
- 按钮：「返回复核」「确认并导出」。
- 点击"确认并导出"时把未确认候选按当前状态真正回写后端确认，并写入**人工确认审计**（来源 = human），不得伪造决策来源。
- 验证：合成 DOCX 走完整流程，弹窗数字正确，导出生成原格式脱敏副本 + 残留复检通过；audit 有人工确认记录（贴 DB 行/文件）。

### A7. prepare 接收并审计日期策略；DATE/TIME 同控
- 根因：prepare 硬编码 `--level strict`，未把设置里的日期开关传给引擎；命中日期的是 strict profile 的 `TIME`。
- 修：
  - prepare 接收 `rulesConfig`（含日期开关），**日期关闭时同时让引擎保留 `DATE` 与 `TIME`**（`--entity-policy preserve_types:[DATE, TIME]` 或等价方式）。
  - **不修改 legal-desens 的 strict 默认语义**（不全局降级）；策略由工作台按需下传。
  - 把本次 `rulesConfig` 写入 audit。
- 默认：**日期保留**（关闭）。
- 验证：日期关闭 prepare → 候选里无 DATE/TIME；日期开启 → 出现完整日期候选（配合任务 B）。贴两态对比。

### A8.（加固）prepare 产物与 DB 一致性
- 文件系统与 SQLite 无法真正同一事务。按此顺序：**临时目录生成 → 完整校验 → 原子重命名到目标 → DB 事务提交 → 失败补偿清理**（任一步失败回滚/清掉半成品，状态置 `failed`）。
- 这是加固项，非当前主因。

> 备注：此前已要求的 UI 清理（去掉"待复核"徽章、"待校对"字样、底部提示条、划词"脱敏"按钮上下 padding 缩短、按类型部分掩码显示 `maskValue`）若尚未全部落地，一并补齐并保持。

## 任务 B · 日期检测规则（legal-desensitizer）

> 仓库：`/Users/clukay/Program/lawchers-skills/legal-desensitizer`
> **不得改 strict profile 的默认语义 / 不做全局降级**。只修"检测到日期时只命中年份"的不完整匹配，以及金额误匹配。

### B1. 日期开启时必须**整体命中完整日期**，不退化成只命中年份
- 修：让日期检测（strict 的 TIME / 相关正则）对以下格式**整体命中**并固定为回归样例：
  - `2026年6月20日` / `2026 年 6 月 20 日` / `2026-06-20` / `2026/6/20` / `2026年6月` / `2026年`
- 真实日期格式未知，先覆盖上面 6 种，拿到真实样本再补。

### B2. 金额不得误匹配日期
- `money_cn` 收紧：金额必须带货币锚点（`¥/￥/元/圆/块/万元/亿元/人民币`），不匹配 `年/月/日` 上下文纯数字。

### B3. 验证（禁止只跑测试名）
- 含 6 种日期 + `15000元` 的合成文本跑 `redact --regex-only`；贴 map.json：日期 start/end 覆盖完整日期、金额仍 MONEY、日期不被误吞；前后对比；跑已有测试无回归。

## 执行规约（强制）

1. **验证铁律**：禁止只用 build/lint / 测试名充验证；涉及引擎/落库/接口/导出，必须贴真实运行输出（CLI 参数、返回 JSON、DB 行、生成文件、截图）。两态对比贴两次并指出差异。
2. **提交铁律**：Conventional Commits；**原子提交**，每个提交自身可构建；**两个仓库分开提交**。在当前分支，不 `git push`。
3. **范围铁律**：只动本任务相关文件；不改产品方向/数据模型/对外 API（先问）；**不改 legal-desens strict 默认语义**。
4. **不并发**：同一仓库同一时间只允许一个 agent 改。任务 A、B 不同仓库，可分别派发。
5. **数据安全**：敏感数据仅存本地（gitignore）；不提交 `*.db`、`uploads/`、`map.json`、`.env`、真实材料。
6. **审计真实**：人工确认/导出写 audit 时来源必须真实（human），不得伪造决策来源。
