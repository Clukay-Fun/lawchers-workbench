# 开发计划 23 · P2 完整编辑模型（多段 diff + 待重选 + 实时高亮）

> 接续 docs/22 的 P2 剩余项。P1/P1.5 已收口。本计划完成"完整编辑模型",但**避开 contenteditable**(中文 IME/光标/粘贴雷区),走"textarea + 高亮层实时重算"的安全路线。
> 定性:收敛 + 纯算法为主,不引新框架。

## 现状（已核）
- 编辑机制:点"编辑原文"→ textarea + 背后一层**静态**高亮(aria-hidden,不随输入移动)→ 点"保存原文"时对全文跑**单区间** LCP/LCS diff(VisualMaskPage `handleTextChange` 1028、`saveEdit` 448)。
- 已完成(P2 部分):编辑后工作文本 + 实体位置持久化(`updateEditedText` / `edited-text.json`,刷新不丢,已 API 回归)。
- 缺口:① 单区间 diff 在"一次编辑改了多处"时把中间整段算成一个变更区 → 中间实体被误取消;② 无歧义处理;③ 高亮不随输入实时移动。

---

## S1 · 完整多段 diff（替换单区间,纯算法,可真测）
- 把 `handleTextChange` 的单前缀/后缀策略升级为**多段 diff**(如 Myers/LCS 分段或等价 token diff),对旧→新文本求**多个变更区间**。
- 实体映射规则(沿用 P1 语义,逐段套用):
  - 落在所有变更区间**之外**的实体:按其右侧累计 delta 平移 offset,**id 不变**。
  - 与**任一**变更区间相交的实体:取消(进 cancelled set)。
- **必须保留不可变 id**(P1 的 `ent_N`),平移只动 offset。
- **verify(像 P1 那样算法层真测)**:一次编辑改两处相隔位置 → 两处之间的实体**保留且 offset 正确**、只有真正相交的实体被取消;贴真实用例证据。

## S2 · 重复文字"待重新选择"（安全网）
- 当某实体在 diff 后**无法唯一定位**(如变更区有重复文本、映射歧义)→ 标记该实体 `needsReselect`,**不猜测关联**。
- UI:该实体在右栏/列表标记"待重新选择",提示用户重新滑选;不自动当作命中、也不静默丢弃。
- **verify**:构造歧义编辑(重复文字附近改动)→ 受影响实体被标 `needsReselect`、未被错误平移到别处。

## S3 · 高亮层实时重算（textarea 路线,不碰 contenteditable）
- 保留现有 textarea;把背后高亮层从"静态原始位置"改为**随每次输入实时重算**:用 S1 的 diff 把当前 `draftText` 的实体位置实时投影到高亮层(高亮层已存在且与 textarea 同步滚动)。
- 编辑中高亮**跟随输入移动**;相交实体实时变灰/标记待重选。
- **明确不做 contenteditable**(中文 IME 合成、光标、粘贴风险)。
- **verify**:编辑中打字,高亮实时跟随、不错位;中文输入法连续输入不卡死、不跳光标(真机手测贴录屏/截图)。

## 边界 / 不做

- 不改 P1 的不可变 id、取消持久化、P1.5 画框反查(已收口,复用)。
- 不做协同/撤销栈等编辑器级功能。
