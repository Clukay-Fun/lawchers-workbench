# 开发计划 13 · 视觉脱敏 UI 收敛（P4，修接线 + UI 状态）

> 背景：docs/12 P1 落地后暴露一批 UI/交互问题。根因集中在一句话：
>
> **遮蔽框 ≠ OCR 框。遮蔽框 = candidates（带 entityType）+ 公章候选 + 强制脱敏词 + 用户手动框。裸 OCR 框不进遮蔽框。**
>
> 即不是"识别能力差导致全页脱敏"，而是 **UI 把 OCR box 当 redact box 喂进去了**。这轮只修接线与 UI 状态。

## 边界（钉死）
- **不碰引擎**（OCR/渲染/导出底层不动）。
- **不做公章识别率增强**（只改文案 + 公章候选可编辑），增强单独排 P4.5。
- 先修接线和 UI 状态，**不引入大框架**。
- 每一步有可验证结果（Karpathy：goal-driven）。

## 两套布局（模式切换 = 切换整个工作区，不是只换右栏）
- **遮蔽模式（默认）**：图像双栏。左 = 原 PDF 页面 + 可编辑遮蔽框；右 = 遮蔽预览 PDF 页面。
- **星号 / 占位模式**：文本双栏。左 = 抽取原文（命中高亮）；右 = 替换后文本预览。导出 TXT/MD/DOCX。
- **不在 PDF 图像上盖星号/占位文本**（两条管线不搅一起）。

## 导出菜单（主按钮只叫「导出」，点开下拉）
- 遮蔽模式：**导出 PDF（遮蔽版）** / 导出 DOCX / 导出 TXT / 导出 MD
- 星号 / 占位模式：导出 DOCX / 导出 TXT / 导出 MD
- **不提供"导出源文件 / 未脱敏原件"入口**（法律数据安全）。

---

## 七项改动（每项带验证）

1. **分离 OCR boxes / redaction boxes（第一刀，根因）**
   - 遮蔽框来源 = candidates(entityType) + 公章候选 + 强制脱敏词 + 手动框；裸 OCR 框不进遮蔽框（最多当可选提示或不画）。
   - verify：上传一份多文字页，遮蔽模式**只黑掉敏感命中**，不再全页黑。

2. **三模式布局切换**
   - `mode` 单一来源；active 样式 / 导出菜单 / 工作区布局 / 预览全绑 `mode`。
   - verify：点 遮蔽/星号/占位 三个 tab **真切换**（之前点星号/占位不切换的 bug 消失）；遮蔽=图像双栏，星号/占位=文本双栏。

3. **连续纵向 PDF 双栏 + 同步滚动 + 同尺寸**
   - 共同滚动容器，一行 = 左页 + 右页；左右用同一 `displayWidth/Height`；滚轮自然翻页；顶部页码 `当前/总`，箭头降级为辅助。
   - verify：向下滚动自然切页；左右同尺寸；滚动同步不漂；不串页。

4. **sticky 工具栏 + 导出下拉**
   - 左侧预览区顶部 sticky：模式切换 / 格式（页内）/ 页码跳转 / 保存框 / 换文件；导出按钮在右上**独立不动**，点开按 `mode` 给格式（见上）。
   - verify：滚动时左侧工具栏吸顶；导出下拉项随模式变；主按钮文案为「导出」。

5. **框交互修正**
   - 选中/hover 蓝色边框 + 很淡的蓝色透明填充（**不变红**）；四角 resize handle 保留；删除 `×` 移到框右上外侧 8–12px、z-index 高于 handle；小框时不压住角点。
   - verify：hover 蓝色、删除按钮不被角点遮、可拖动/缩放/删除。

6. **上传 / 处理进度**
   - 上传：真实百分比（XHR upload progress）。处理：步骤式（上传中 → 渲染页面 → OCR → 规则匹配 → 生成预览），**不伪造百分比**。
   - verify：上传大文件能看到真实%；处理阶段有步骤反馈。

7. **文件名规范**
   - 导出名 = `原文件名_脱敏.ext`；**不加 `.placeholder`/`.star`** 等后缀；重名可加时间/序号但下载名不出现这些标记。
   - verify：三模式各导出，文件名为 `xxx_脱敏.pdf/docx/txt/md`。

8. **公章文案（仅文案）**
   - 改为「已识别公章候选，可手动补充或调整」；公章候选作为可编辑框，检测不到时用户可手动框。

## 验收
- 遮蔽模式不再全页脱敏（#1）。
- 三模式真切换、布局正确（#2）。
- 纵向滚动翻页、左右同尺寸同步滚动（#3）。
- 工具栏吸顶、导出下拉按模式、主按钮叫「导出」、无"导出源文件"入口（#4）。
- 框 hover 蓝色、删除按钮不被遮、可拖缩删（#5）。
- 上传真实进度 + 处理步骤（#6）。
- 文件名 `_脱敏` 无 `.placeholder`（#7）。
- 全程不碰引擎、不引大框架。每条贴真实证据（截图/导出文件）。

## 约束
- 只动前端 UI 接线与状态（必要的后端仅文件名/导出格式路由），不动 OCR/渲染/导出引擎逻辑。
- 验证铁律：禁止只 build/lint，贴真实截图与导出文件。
- Conventional Commits、原子提交、当前分支、不 push。
